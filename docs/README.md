# Documentation

These docs are the project-specific source of truth for agents and contributors. They are intentionally concise so they stay useful during day-to-day development.

## Map

- `architecture.md`: app structure, major modules, and aliases.
- `development.md`: setup, scripts, environment variables, and deployment notes.
- `testing.md`: validation strategy and test locations.
- `design-system.md`: UI, Tailwind, component, and accessibility conventions.
- `content-model.md`: where portfolio content and public assets live.
- `agent-workflow.md`: how AI agents should approach tasks in this repo.

## Maintenance Rules

- Update docs in the same change when commands, routes, env vars, or architecture change.
- Prefer examples from this codebase over generic framework advice.
- Keep each page focused. Link to a nearby doc instead of duplicating large sections.
